# Nomi v0.16.2

Patch release that makes image-to-image (reference-based) generation actually honor your source images.

## Highlights

- Legacy relay image models are auto-upgraded on launch (catalog v5 migration): the image-edit transport, reference slots, and 1K/2K/4K resolution options now appear without deleting and re-adding the model.
- Reference URLs now prefer the local persistent copy of each asset; expired provider CDN links (most last only 1–3 days) are replaced by re-uploading the local bytes automatically, so references no longer rot when a project is reopened later.
- Honest guardrails: image-edit / image-to-video requests with no usable reference — or a model with no image-edit channel — are rejected **before any spend** with a clear message, instead of silently degrading to text-to-image. The generate button is disabled with guidance when i2i has no reference, and stale reference chips now read "图已失效" with a recovery hint.

## Validation

- 2548 unit/integration tests, including new coverage for the catalog v5 migration, URL priority, the 48h freshness window, refuse-before-spend guards, and a wire-level check that references actually reach the outgoing request body.
- Real-app Playwright walkthrough: on-disk v4→v5 migration, reference slot + resolution appearing for a legacy relay model, the disabled-generate guardrail, and the invalid-reference chip.
- Typecheck, lint, unit tests, and production build passed.
