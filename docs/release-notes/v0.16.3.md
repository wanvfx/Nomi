# Nomi v0.16.3

Patch release: honest, immediate errors for Jimeng (Dreamina) CLI video generation, an Agnes image-to-video fix, and the Windows 3D fullscreen window bar.

## Highlights

- Jimeng CLI video tasks no longer hang in "still generating" for 8-15 minutes when the CLI silently fails (expired login, first-use web authorization, non-VIP account): the node now fails within seconds with the real reason and what to do. These errors are also no longer misclassified as "model not activated (Volcano Ark)".
- Agnes image-to-video: the first-frame parameter was sent as an array while the backend expects a single URL string; the archetype's single-image declaration is now honored.
- Windows: fullscreen 3D scene now shows a window bar (logo + minimize/maximize/close) above the scene toolbar instead of covering the title bar. Thanks to @1251912798 (partial merge of PR #33; the in-app browser asset capture is being reworked to a leaner spec).

## Validation

- New unit coverage: dreamina silent-failure parsing ("no id / no status / no media = fail"), error-classifier ordering, Agnes asArray input shape.
- Real-app walkthroughs: live dreamina failure injection (error card within 3s, correct "账号权限不足" title), darwin 3D fullscreen regression (header intact, no win32 bar leak).
- Typecheck, lint, unit tests, and production build passed.
