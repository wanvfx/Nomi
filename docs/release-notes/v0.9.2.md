# v0.9.2 — 修文本生成"没找到对象"

发布日期：2026-06-04

## 一句话

**修好文本生成在 Base URL 带 `/v1` 时报"没找到对象"。**

## 修复

- **文本生成 URL 双 `/v1`**：当模型 Base URL 以 `/v1` 结尾（如 Moonshot 官方 `https://api.moonshot.cn/v1`、README 也这么教），文本生成会把地址拼成 `…/v1/v1/chat/completions` → 供应商返回"没找到对象"。现在 base 填不填 `/v1` 都正确（5 例单测锁定）。图片生成不受影响。

## 内部

- runtime 模块化拆分（jsonUtils / tasks / assets / catalog / runtimePaths / projects 等纯层抽出 + 配套测试），降低单文件体量、提升可维护性。测试总数增至 513。

> v0.9.0/v0.9.1 用户：若用 Moonshot 等 Base URL 带 `/v1` 的模型做文本/拆镜头，请更新到 v0.9.2。
