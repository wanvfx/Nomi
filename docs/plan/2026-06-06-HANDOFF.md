## 3. 已定方向（别推翻，用户已拍板）

**通用「素材引用」系统**（= 不是 Seedance 专用）= 3 原语 + 2 规律，住共享模块 `src/workbench/assets/`：
- **AssetTile**：正方形、形态自明（图缩略图 / 视频缩略图+播放三角+暗蒙 / 音频波形）。
- **AssetPicker**：统一添加（画布 / 项目素材[搜索+最近+浏览全部] / 上传 / 拖入 / 连线），一个「+」点开就是它。
- **AssetMention**：描述里 `@` 或**点 tile** → 缩略图进句子；**发送前才转 `character1..N`，用户永不可见 character1**。
- 规律：**快速取（弹层搜+最近）vs 全量浏览（素材面板）**；**形态自明 > 文字解释**。
- **别混淆**：P1 的 `AssetReference.tsx` **不是第 4 个通用原语**——它是**节点侧的 slot 组件**（吃档案 slots、消费上面 3 原语、替换旧 `ReferenceSlots`），住节点 `controls/`，**不进** `assets/` 共享模块。通用原语永远只有 3 个。

**完整用户旅途（整个架构的收口）**：① 接入（放文档+API）→ ② 识别分层（**一切皆档案**：认得用精修档案 / 认不出用
文档**派生**的档案，**永不裸奔**）→ ③ 统一使用（节点=档案+原语组装）。关键：flat 启发式从「渲染兜底」翻转成
「**接入时派生引擎**」→ UI 端永远只认档案。

## 5. 关键约束 / 必踩清单（6 角色评审交叉命中，**执行前务必读**）

- **R1（最致命，会让真实生成失败）`nomi-local://` 素材 vendor 取不到**：上传/项目文件素材 URL 是 `nomi-local://`，
  runtime 把参考 URL 原样发 vendor、**无 local→remote 上传**。统一素材池一上就炸。**P0/P1 定契约**：`AssetRef` 区分
  「渲染 URL」vs「传输 URL」；**发送前在 runtime 侧**把 nomi-local 上传成 vendor 可达 URL（走已有 `hardenedFetch`
  私网拦截 + 200MB 上限）。证据：`electron/runtime.ts`（`writeAsset`/`importRemoteAsset`/`taskTemplateParams`）。
- **R-传输（别动中间）**：§2 那条传输单源链是「代码即合同」，重构只能动两端（UI 原语在上、vendor body 在下），
  **绝不在中间另起参考组装路径**。
- **R2 P4 拆分 + 不落库**：派生档案**按需算+进程缓存、不入库**（落库=陈旧档案债=第二真相源，违背总纲）。拆
  **P4a**（`deriveArchetypeFromCatalogMeta` 纯函数，内部调现有 `parameterControlModel` 启发式）/ **P4b**（缓存，可缓做）。
  **硬门 = 逐键等价快照测试**：派生档案产出的 archetypeInput/slots/params 必须和旧 flat 路径**键级相等**；尤其删
  `catalogTaskActions.buildReferenceExtras` 的 flat else 兜底前，否则老模型首尾帧**静默丢传**。删的永远是「UI 第二条路径」，
  **保的是解析/派生能力本体**（别把派生引擎当死代码误删→认不出的模型会裸奔）。
- **R3 必补样张（先出样张再写码）**：P5 满配节点合成（四原语同屏垂直节奏=验收合同）、P4 派生档案（真实「认不出模型」
  渲染，证明不是 snake_case Swagger）、P2 ModelPicker、P3 文本节点。
- **R4 派生档案视觉兜底**：参数 >N 折叠进「高级」、snake_case 展示层美化（title-case，真名留传输层）、出样张审。
- **R5 AssetReference 声明式 slot 描述符**：slot 声明 `{cardinality:single|array, persistAsEdge, form:tile|textRef|singleImage}`，
  原语按它分叉渲染。**数组绝不变持久边**（崩 `(target,mode)` 唯一性、回归全能参考；M6 的 meta-only 要保留）。
  文本节点「选中文本」槽是 `textRef` 形态、不是 tile。
- **R6 @ 编号单源 + Tiptap**：@ 投影（prompt 文本）和参考槽有序 URL 数组**在同一纯函数一次产出**（否则两套编号漂移），
  复用 `archetypeMeta` 已有的 `characterIndexed` ①②③ 语义。**Tiptap 能做、不用 spike**（官方 inline atom node +
  `ReactNodeViewRenderer` + Mention/suggestion，且 Tiptap 已是项目依赖、已在用 `WorkbenchEditor`/`useNomiRichTextEditor`）。
  prompt 现在是纯 `<textarea>`（NodeGenerationComposer），换 Tiptap 是真替换（换了删旧、别两套并存）。
- **R7 P2 最安全**：只删**写侧**双轨（`imageModel/videoModel`），**读侧保留 `||imageModel/videoModel` 兜底 + legacy backfill**，
  0 migration；`adapters/modelOptionsAdapter.ts` 实测 **13 个死导出可删**。runtime（catalogTaskActions）已 `modelKey` 优先。
- **assetPool 别建新 store**：用 useMemo 派生 selector 合流（画布 store 切片 + `useWorkspaceFiles`，后者已 limit 500），
  Map 去重 O(n)、最近切前 N、全量走虚拟滚动；否则就是「第二份真相源」。
- **节点级 onDrop**（P0.4）：现在没有；加时要 `onDragOver+preventDefault+dropEffect='copy'` + `stopPropagation`
  （否则要么 drop 不触发、要么既加参考又被画布 onDrop 建新卡）；三种 drag payload 共用一套解析。

**🔴 旅途盲点（真实用户最痛，独立并行轨，优先级高于节点重构）**：整条路最劝退的是第一步**「贴接入文档 URL」**——
非技术创作者做不到、进门就走。用户要：**模型列表直接点（别贴 URL）/ 最多填 key（配截图）/ 先空手玩起来尝甜头**。
接入页 = `src/ui/onboarding/`（`OnboardingWizard`/`OnboardingDrawer`/`OnboardingFloatingPanel`）。**建议作为并行轨先评估**。
徽标：模板/通用/角色/vendor 真名多套 → design system 登记一套 pattern；**精修档案不打标、只克制标派生**，别逼用户懂内部两档。

## 6. 工程纪律（强制，违反即工作错误 —— 全在 `CLAUDE.md`，每会话必读）

- **全绿 ≠ 完成**：CI 五门只证明代码健康，**catch 不到设计保真/体感**。用户可见改动报「完成」前，必须 ①
  **真渲染样张并排逐项对账**（规则 8 AFTER 步，不是拿脑中印象）② **真体感走查**（规则 13，看截图比样张，不是 `expect(元素存在)`）。
- **设计/UI 工作自己主动跑评审**：动 UI/定方向时**自动**起设计师 + 真实用户 agent 审（规则 7/8），不等用户点名。大方案过 6 角色。
- **好产品不靠解释**：UI 一堆名称/说明=设计没做好；优先形态自明，文字降到最少。
- **通用第一（能力层 + 交互层）**：档案按模型身份认、与供应商解耦；UI/交互原语通用、谁用谁声明槽——别给单模型/功能写专用版（=规则 1 并行版）。
- **加新必删旧**（规则 1）；**先想模块化/架构**（规则 9）；**碰库先查官方文档**（规则 5，Context7 或 WebFetch）+ **读顶尖开源**（规则 6）；
  **歧义/矛盾必停上报**（别自己挑一条）；**文件 ≤800 行、巨壳白名单只减不增**（规则 12，`pnpm run check:filesize`）。
- **每个完整改动自己 commit + push**（规则 11，结尾带 `Co-Authored-By`）；**push 前跑全套 CI 门**（见 §7）。
- **Context7 可能没接通**：R1（`hardenedFetch`）、R6（Tiptap）都触发规则 5「碰库先查官方文档」。但 Context7 MCP 常因项目级 `.mcp.json` 未审批而**不可用**（`✓ Connected` 只是可达性、不代表工具可调）——若 `ToolSearch "resolve-library-id"` 找不到工具，跑 `/mcp` 批准，或**直接退回 WebFetch 查官方站**（`tiptap.dev` 等，等价），别卡在审批上。
