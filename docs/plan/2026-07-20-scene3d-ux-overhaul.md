# 3D 导演台 UX 大改：从功能堆砌到用户旅程

> 日期：2026-07-20
> 状态：**P0 全部（出片按钮/面板/已有轨迹直出/产物卡/接力 toast）+ P1-6/9（5 步引导·带兜底锚点/重看）
> + P2-10/12（预设提顶/时间轴默认开）+ P3-16/17 已落地并 R13 走查全绿**（`scripts/scene3d-export-journey-walkthrough.mjs`，
> 截图 `.scene3d-ux-lab/`）。落地时补了三个根修：① 出片时长裁到运动终点（默认时间轴 10s ≠ 轨迹时长，不裁
> mp4 带定格尾巴）；② 引导第 4 步 fallback 锚点（运镜面板未渲染时退指相机行，否则静默跳步）；③ demand
> 帧循环下暂停拖播放头补 invalidate（时间轴常显后不再靠 timelineOpen 强制 always，不补则拖动视口冻住）。
> **P2-12 当日修正（用户真机反馈「一进来是时间轴，懵逼；关掉才见添加」）**：本方案里"时间轴默认开"
> 与 P2-13"添加移左栏"是一对——底部被时间轴占了添加就得搬家。只落前一半 = 时间轴盖住「添加」，
> 旅程第 1 步（摆场景）被挡。修正为**时间轴按旅程时机出现**：默认收起，预设落轨 / 进轨迹模式 / 绑定 /
> 播放任一发生即自动打开；P2-13 撤销（添加留底部，待 P2-11 布局重组一起看）。同轮补 P1-7 空态指路：
> 右栏未选中时顶部旅程提示卡 + 时间轴空态改「下一步」话术。走查补走真实第 1 步（添加→城市街道）防复发。
> 剩余：P2-11 整运镜三 tab 合一、P3-14/15——属布局大改，按 R8 出样张拍板后再做。
> 起因：用户反馈"3D 场景功能多但不会用，整轨迹后不知道产出什么、没有导出入口、交互路径混乱"
> 范围：3D 场景全屏编辑器（Scene3DFullscreen）的 UI/UX + 功能逻辑 + 交互流程
> 不改：3D 渲染引擎、Scene3DState 数据结构、与画布节点的接线协议

## 0. 一句话定调

**3D 场景当前是"功能堆砌"，要改成"以出片为锚点的用户旅程"。** 用户进来的唯一目标是拿到参考视频/截图喂给下游镜头——所有功能都应该服务于这条路径，而不是平行展开让用户自己拼。

## 1. 问题诊断

### 1.1 当前功能清单（用户视角）

| 功能 | 入口 | 用户能干什么 |
|---|---|---|
| 假人控制 | 左侧选中假人 → 顶部"操控" | 换姿势、WASD 走位、动作库 |
| 相机控制 | 左侧选中相机 → 顶部"操控" | WASD 飞、转朝向、滚轮推拉 |
| 运镜预设 | 选中相机 → 右侧属性 → 滚到底部 | 13 招一键落轨迹 |
| 场景模板 | 底部"添加" → 场景模板 | 城市街道/室内房间 |
| 道具 | 底部"添加" → 道具 | 车/楼/树/几何体/灯光/假人 |
| 轨迹 | 顶部"轨迹"按钮 / 底部"轨迹" / 右侧"轨迹"tab / 双击空地 | 加点、拖点、绑定、时间轴 |
| 录 take | 选中假人/相机 → 操控 → 底部 REC | 实时操控录制 → 出 mp4 |
| 截图 | 顶部"截图" / 相机预览"相机截图" / 运镜首尾帧导出 | 三种截图方式 |

### 1.2 五个用户旅程断点（核心问题）

**断点 1：进了不知道第一步干什么**
- Coach Marks 只出现一次，且只指"点假人/点相机/添加"三个孤立动作
- 没讲"摆场景 → 摆角色 → 摆相机 → 整运镜 → 出片"的整体流程
- 第二次进入零引导

**断点 2：整了轨迹不知道怎么播放**
- 时间轴默认不显示（要进轨迹模式或绑定后才开）
- 没绑对象直接播放会被 toast 拒绝，但用户不知道要先绑
- 播放按钮在顶部和时间轴两处，但都不显眼

**断点 3：整完轨迹不知道怎么出参考视频 ★ 核心**
- **没有"导出参考视频"按钮**
- 出 mp4 的唯一入口是"录 take"，藏在：选中假人/相机 → 点"操控" → 底部条 REC
- 走"轨迹"路径（加点、拖点、绑相机）整完后，**没有任何入口能把轨迹导出成视频**
- 运镜预设能生成轨迹，但不会自动出 mp4

**断点 4：出片后不知道产物去哪了**
- mp4 异步生成，全屏编辑器盖着画布
- 关掉编辑器才看到新节点（fitView 推迟到关闭后执行）
- 生成中徽标只在源节点底部显示，用户不回画布看不到

**断点 5：核心功能入口藏太深**
- 13 招运镜预设要"选中相机 → 右侧属性面板 → 滚到底部"才出现
- 录 take 要"选中 → 操控 → 底部条"
- 轨迹编辑要"进轨迹模式"才切换左侧面板

### 1.3 根因

**功能驱动而非旅程驱动。** 每个功能单独看都做完了（有入口、有交互、有反馈），但没有人把它们串成一条"从用户目标到产物"的路径。用户面对的是 7 个并列的功能区，不是一条流程。

## 2. 设计原则

### 2.1 逆向设计：从出片倒推

不再"有什么功能就摆什么入口"，而是**先定用户要拿什么产物，倒推需要什么步骤**：

```
产物（参考视频/截图）
  ↑
出片动作（显式按钮）
  ↑
运镜就绪（轨迹/预设/录 take 任一）
  ↑
相机就位（位置+画幅+焦段）
  ↑
角色就位（姿势+位置）
  ↑
场景就位（模板+道具）
  ↑
用户进入
```

每一步都是下一步的前置条件，每一步都告诉用户"下一步干什么"。

### 2.2 三个设计不变量

1. **出片是显式动作**——顶部工具栏永远有"出片"主按钮，不是藏在某个功能里
2. **每一步有"下一步"提示**——空状态、完成态、接力 toast 三种引导
3. **产物永远可见**——出片后在编辑器内显示产物，不用关掉编辑器找

### 2.3 不动的边界

- 不重写 3D 渲染引擎（@react-three/fiber + drei）
- 不改 Scene3DState 数据结构（向后兼容已有项目）
- 不改与画布节点的接线协议（scene3d 节点 → 截图/参考视频节点 → 下游镜头）
- 不取消任何现有功能（只是重新组织入口和流程）

## 3. 目标用户旅程

### 3.1 完整 5 步流程

```
Step 1: 进 3D 场景
  ↓（首次有 5 步引导，非首次有空状态提示）
Step 2: 摆场景（模板 + 道具 + 角色）
  ↓（场景就位后提示"下一步：摆相机"）
Step 3: 摆相机（位置 + 画幅 + 焦段）
  ↓（相机就位后提示"下一步：整运镜"）
Step 4: 整运镜（预设 / 轨迹 / 录 take 三选一）
  ↓（运镜就绪后提示"下一步：出片"）
Step 5: 出片（参考视频 / 截图 / 首尾帧）
  ↓（产物在编辑器内可见，一键喂下游）
```

### 3.2 每一步的入口、动作、产出

| 步骤 | 入口 | 用户动作 | 产出 | 下一步提示 |
|---|---|---|---|---|
| 1 进 3D | 画布 scene3d 节点 | 点"打开 3D 编辑器" | 空场景 + 引导 | "先选个场景模板" |
| 2 摆场景 | 底部"添加" | 选模板/加道具/加假人 | 场景里有东西 | "加个相机才能出片" |
| 3 摆相机 | 底部"添加" → 相机 | 拖位置/调画幅/调焦段 | 相机就位 | "整运镜才能出参考视频" |
| 4 整运镜 | "整运镜"入口（三选一） | 选预设/画轨迹/录 take | 运镜就绪 | "点出片生成参考视频" |
| 5 出片 | 顶部"出片"主按钮 | 选产物类型 | mp4/截图/首尾帧 | "已生成 → 喂给下游镜头" |

### 3.3 三种运镜方式的统一入口

当前：轨迹 / 运镜预设 / 录 take 三个入口分散在不同地方。

改后：合并成一个"整运镜"入口，点开给三个 tab：

```
┌─────────────────────────────────┐
│  整运镜                          │
│  ┌─────┬─────┬──────┐          │
│  │预设 │轨迹 │录 take│          │
│  └─────┴─────┴──────┘          │
│                                 │
│  [当前 tab 的内容]               │
│                                 │
│  ✓ 运镜就绪后 → [点这里出片]     │
└─────────────────────────────────┘
```

- **预设 tab**：13 招运镜一键落轨迹（核心卖点，第一屏可见）
- **轨迹 tab**：手动加点/拖点/绑定
- **录 take tab**：实时操控录制（保留现有 REC 流程，但入口提到这里）

## 5. 功能逻辑改造

### 5.1 出片动作统一化

**当前**：三种产物三个入口（截图在顶部、首尾帧在运镜面板、mp4 在录 take）

**改后**：统一到"出片"主按钮的出片面板，内部调三个函数：
- `exportReferenceVideo()`：沿轨迹渲染 mp4（复用现有 CameraMoveCaptureHost 逻辑）
- `exportScreenshot(type)`：视口/相机取景截图（复用现有 handleScreenshot）
- `exportKeyFrames()`：运镜首尾帧（复用现有运镜导出）

**关键改动**：`exportReferenceVideo()` 要支持"已有轨迹直接渲染"，不只是"录 take 后渲染"。当前只有录 take 才会触发 CameraMoveCaptureHost，要改成：有轨迹 + 有绑定相机 → 就能直接渲染。

### 5.2 运镜就绪状态

新增一个"运镜就绪"判断：
```ts
function isCameraMoveReady(state: Scene3DState): boolean {
  return state.trajectories.some(
    t => t.bindingNodeId && getNodeById(t.bindingNodeId)?.type === 'camera'
  );
}
```

- 运镜就绪时：出片按钮高亮 + 右侧"整运镜"区底部出"点这里出片"接力按钮
- 运镜未就绪时：出片按钮灰掉（但可点击，点击后提示"先整运镜"）+ 出片面板里"参考视频"灰掉

### 5.3 接力提示系统

在关键节点出接力 toast/banner，引导用户走下一步：

| 触发条件 | 提示内容 | 动作 |
|---|---|---|
| 场景里有东西 + 没相机 | "场景就绪 → 加个相机才能出片" | [加相机] |
| 相机就位 + 没运镜 | "相机就位 → 整运镜才能出参考视频" | [整运镜] |
| 运镜就绪 + 没出片 | "运镜就绪 → 点这里出参考视频" | [出片] |
| 出片完成 | "参考视频已生成 → 喂给下游镜头" | [喂下游] |

接力提示用 toast + action button 的形式，用户可以一键跳到下一步。

### 5.4 引导系统重做

**当前 Coach Marks**：3 步（点假人/点相机/添加），只出现一次

**改成 5 步流程引导**：
1. "先选个场景模板" → 指向底部"添加" → 场景模板
2. "加个角色" → 指向底部"添加" → 假人
3. "加个相机" → 指向底部"添加" → 相机
4. "整运镜" → 指向右侧"整运镜"区
5. "出片" → 指向顶部"出片"按钮

**关键改动**：
- 每步做完自动进下一步（不是只指一下就完）
- 角落加"重看引导"按钮（不只第一次）
- 可跳过（保留现有 skip）

### 5.5 空状态提示

每个区域没内容时给空状态 + 下一步提示：

| 区域 | 空状态提示 | 动作按钮 |
|---|---|---|
| 左侧场景元素树 | "场景是空的，先加点东西" | [加场景模板] [加假人] [加相机] |
| 右侧整运镜区 | "还没整运镜，选个方式开始" | [选预设] [画轨迹] [录 take] |
| 底部时间轴 | "整运镜后这里能预览效果" | (无) |
| 出片面板（没运镜时） | "先整运镜，才能出参考视频" | [去整运镜] |

## 6. 交互细节

### 6.1 出片流程的状态机

```
[未出片]
   ↓ 点"出片" → 出片面板
[出片面板打开]
   ↓ 选"参考视频" → [生成]
[生成中]（右下角进度卡片）
   ↓ 生成完成
[生成完]（右下角产物卡片）
   ↓ 点"喂给下游镜头" → 自动接线 + 关编辑器 + fitView
[已出片]
```

### 6.2 预览（播放）逻辑

- 顶部"预览"按钮 = 播放当前运镜看效果（不出片）
- 没运镜时灰掉，hover 提示"先整运镜"
- 没绑对象时灰掉，hover 提示"轨迹要先绑定对象或相机"
- 播放时按钮变"⏸ 停止"，再点停止

### 6.3 错误提示优化

| 场景 | 当前提示 | 改后提示 |
|---|---|---|
| 没选相机点截图 | "请先选中一个拍摄相机" | "要先选中一个相机才能截图 → [去选相机]" |
| 没绑对象点播放 | "请先为轨迹绑定对象或相机" | "轨迹要先绑定对象或相机才能播放 → [去绑定]" |
| 录 take 没动 | "没录到走位，请操控角色走动后再录" | "你刚才没动过——WASD 走位/运镜才会被录进去，再试一次？" |
| 单场景超 100 对象 | "单个 3D 场景最多支持 100 个对象" | "场景满了（上限 100 个对象），删掉不用的再加新的" |

每个错误提示都带 action button，一键跳到解决入口。

### 6.4 产物去哪了

出片后产物有三个去向，都要让用户知道：
1. **画布新节点**：截图/参考视频都建画布节点，位置在源 scene3d 节点右侧/下方
2. **自动接下游**：参考视频自动注入下游镜头的 referenceVideoUrls
3. **素材库**：参考视频也落进项目素材库的 imported 桶

出片完成后，产物卡片里显示：
```
✅ 参考视频已生成
   → 已创建画布节点（参考视频 1）
   → 已接到下游镜头（镜头 3）
   → 已存进素材库
[预览] [跳到画布] [喂给另一个镜头]
```
