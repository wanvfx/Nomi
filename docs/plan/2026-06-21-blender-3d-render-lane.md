# Blender 3D 渲染 lane：用 headless Blender 给 Nomi 补上「跨镜一致 + 真相机控制」

> 2026-06-21。起点：用户想「让 AI 把人类工具（PS / Blender）用起来，最差也能用 3D 建模直接做视频」。
>
> 三路最新调研（2026-06-21，详见对话）后拍板的方向裁剪：
> - **PS 方向放弃驱动 PS**，并入「生成式编辑模型 + Nomi 自有蒙版/合成层」（另议，不在本文档）。
> - **Blender 方向**值得做，但**只做一条窄链**，范围由本文档框死。
>
> 本文档只覆盖 Blender lane。是**探索性后续 lane，不是当前主线**——排期优先级待用户拍（见 §8）。
>
> **✅ 2026-06-21 已做端到端 PoC 实测（用户机器 Apple M5 / 24GB / Blender 5.1.2）——核心链路已证可行，见 §9。**

---

## 0. 一句话战略定位（为什么是 Blender，为什么是这条窄链）

Nomi 的真痛点之一：**扩散视频模型结构上做不到「跨镜头身份/尺寸一致」和「真·相机控制」**——同一个角色第 1 镜和第 50 镜会漂，机位是"祈祷"出来的不是设定出来的。

3D 恰好天然给这两样：同一个 3D 资产在任意镜头渲染必然一致，相机是数值设定的。**这是 3D 对 Nomi 唯一的、结构性的、对手抄不动的价值**——不是"我们也能做 3D"这种广度，而是补一个生成模型补不上的洞。

**但 2026 的硬现实（调研结论，§2）决定了范围必须狠砍**：AI 全自动绑骨/角色动画那堵墙没破。所以本 lane **只做**：

> **AI 生 3D 资产 → headless Blender 子进程布景/打灯/设相机/渲染「简单运动」镜头 → 出视频片段 → 进 Nomi 时间轴。**
> **不碰**自动绑定、角色表演动画、复杂物理。

这条窄链是真能跑通的（§2 有据），且和 Nomi「生成为主」的现有定位是**叠加**不是替换——3D 只负责生成模型啃不动的那几个"需要绝对一致 + 精确机位"的镜头。

## 1. 严格范围 + 不做项（本文档的核心，违反即范围失控）

### 1.1 做（且只做这些）

| 能力 | 说明 |
|---|---|
| **AI 生 3D 资产** | 接 text/image→3D 的现役 API（Tripo / Meshy / Rodin，§2.3）作为新的资产来源，产 GLB/FBX |
| **headless Blender 渲染原语** | 子进程 `blender -b --python`：导入资产 → 布景（位置/朝向/缩放）→ 打灯 → **设相机（机位/焦段/简单推拉摇移关键帧）** → 渲染出视频片段 |
| **跨镜一致** | 同一资产 + 同一/连续相机轨迹复用到多镜，保证身份/尺寸一致 |
| **片段进时间轴** | 渲染产物当普通视频节点喂进现有时间轴，和生成模型片段混剪 |
| **AI 驱动 = 写脚本** | AI 产的是「场景描述 → bpy 脚本参数」，不是操作 GUI（§2.1 死路已排除） |

### 1.2 不做（明令禁止，越界即停下上报）

- ❌ **不做自动绑骨 / 角色表演动画**——2026 这堵墙没破（§2.2），强行做=不可控的烂活。需要角色动起来的镜头**退回扩散视频模型**。
- ❌ **不做复杂物理 / 流体 / 布料 / 粒子**——算力与可控性双输。
- ❌ **不做 GUI 自动化**——不让 AI 看屏幕点 Blender 界面（§2.1，3–17% 成功率，大厂全弃）。
- ❌ **不捆绑 Blender 二进制进安装包**——几百 MB + GPLv3 分发义务 + 跨平台签名/更新痛（§5）。做法是**检测用户已装 + headless 调用**。
- ❌ **不做手动 3D 编辑器**——Nomi 已有的 Scene3D 站位参考（`create_staging_reference`）是"出参考图"，本 lane 是"出渲染片段"，**两者不合并、不互相侵蚀**。
- ❌ **不内置 PS、不驱动 PS**——见文首，另议。

### 1.3 范围自检

> 任何提议若需要「让角色做表演动作」「让 AI 在 Blender 界面里点」「把 Blender 打进安装包」——**立即停，违反 §1.2**。

## 2. 现实约束（2026-06-21 调研，带据；不可凭记忆）

### 2.1 AI 用 Blender 只能走脚本层，GUI 视觉路已死
- 所有真能干活的案例都是 LLM 写并跑 **bpy（Blender Python）脚本**，不是操作 GUI。
- GUI 视觉路：OpenAI 砍 Operator、Google 砍 Mariner；PSBench（ICLR 2026）实测视觉 agent 操作创意软件成功率 3–17%。→ 本 lane 只走脚本层。

### 2.2 全自动出成片动画不存在，绑定/动画是硬墙
- 截至 2026-06，无任何可信、独立验证的「AI agent 从 prompt 自主渲出成片角色动画」案例。
- 已验证能干的只是 **AI 搭静态场景 + 简单物体位移 + 相机运动**；绑骨/复杂动画仍需人。→ §1.2 第一条的依据。
- AI 生的网格多为"三角面汤"，要变形动画得人工重拓扑——本 lane 因此只用资产做**布景/道具/可整体移动的物体**，不做需要变形的角色表演。

### 2.3 text/image→3D 已到「道具/背景/可整体移动资产」生产可用级
- 现役有 API 的：**Tripo**（清四边面口碑最好，auto-rig 出 T-pose）、**Meshy**（全格式 + 内置 auto-rig）、**Rodin/Hyper3D Gen-2.5**（三种拓扑模式，主打免重拓扑，但 API 限 Business+ ~$120/mo）。
- 开源自托管：**Hunyuan3D 2.1**（密三角面、~29GB VRAM）、**TRELLIS**（MIT）。
- 铁律：**基础网格/道具/背景已生产可用；英雄角色/可动角色不经人工清理不可直接用**——再次指向 §1.2。

### 2.4 headless Blender 工程事实
- `blender -b --python script.py` 4.x/5.x 标准支持（渲染农场/CI 常规用法）。
- **跨平台 headless 必须用 Cycles**：EEVEE 在 Win/macOS 无法 headless（需 GL 显示上下文），仅 Linux 自 3.4 可。
- macOS 真二进制路径：`/Applications/Blender.app/Contents/MacOS/Blender`。
- CLI 参数按顺序执行，`-f`/`-a`（渲染）必须放最后；GPU/CPU 选择需烤进 .blend 或脚本里 `get_devices()` 后设，background 下默认静默回退 CPU。
- 渲染成本现实：Cycles 每帧分钟级，VRAM 是硬约束。→ 本 lane 默认**短片段、低帧数、Cycles**，把"慢"控制在可接受范围。

## 3. 目标架构（接进 Nomi 现有骨架，不另起炉灶）

```
  现有: 创作意图 ──► archetype/catalog ──► runtime.runTask ──► vendor HTTP ──► 生成片段 ──► 时间轴节点
                                              (electron/runtime.ts:552 四条分路)

  新增 Blender lane (作为 runtime 的「第五条渲染源」，复用同一产物口径):

    ① 资产生成  ──►  新 archetype kind: "asset3d"  ──►  Tripo/Meshy/Rodin API (复用现有 vendor/catalog 接入机制)
                                                          产物: GLB/FBX 落本地素材库

    ② Blender 渲染原语 (主进程, headless 子进程)
       AI 产「场景描述 JSON」(资产引用 + 布景 + 灯光 + 相机关键帧)
          │
          ▼  组装成 bpy 脚本参数 (模板化, 不让 AI 裸写 bpy → 防 API 漂移/注入)
       child_process.spawn(blenderBin, ['-b', '--python', renderScript, '--', sceneJsonPath])
          │
          ▼  Cycles 渲染 → 视频片段落本地
       片段当普通视频节点喂进时间轴 (与生成模型片段同一口径, 混剪)
```

### 3.1 几条铁律

- **R1（P1 加新必删旧）**：Blender 渲染源是 runtime 的新分路，**不是并行版**——产物口径（落本地 + 进时间轴节点）必须和现有生成片段完全一致，时间轴/导出**零特判**。
- **R2（防 AI 裸写 bpy）**：AI 产的是**结构化场景描述 JSON**（资产/布景/灯光/相机），主进程用**固定模板**组装 bpy 脚本，不让 LLM 直接输出可执行 Python——挡住 §2 的两大风险：bpy 版本 API 漂移、任意代码执行/注入。这与 capability-core 文档「AI 产配置不产代码」一脉相承。
- **R3（headless 子进程，§5 法律安全港）**：只经 `child_process.spawn` 调用，传文件 + CLI 参数，绝不 in-process 链接 bpy、绝不开常驻 socket。
- **R4（资产生成复用现有接入）**：3D 资产 API 当新 archetype kind 接，复用现有 vendor/catalog/providerUrl 那套，不另搭一套接入。

## 4. 分期切片（每片独立可验、可停）

| 切片 | 内容 | 验收 |
|---|---|---|
| **S0 探针** | 检测用户机器是否装 Blender（PATH → 各 OS 默认路径 → `--version` 校验），版本/能力探测；无则引导安装 | 真机三平台探测到/探测不到都给对文案 |
| **S1 最小渲染原语** | 主进程 headless 子进程：导入一个本地 GLB → 单相机 → Cycles 渲一张静帧 | 真机渲出图，落本地，主进程零崩 |
| **S2 资产生成接入** | 接 1 个 3D API（建议先 Tripo，四边面+有 auto-rig+价合理）当 asset3d kind | 真 API 生出 GLB 落素材库（真额度，默认授权直接跑） |
| **S3 场景描述 → 视频片段** | 场景 JSON（布景+灯光+相机关键帧）→ 模板组装 bpy → 渲短视频片段 → 进时间轴 | 真机出一段「资产 + 推拉镜头」片段，进时间轴能混剪 |
| **S4 跨镜一致验证** | 同资产复用到 2+ 镜，验证身份/尺寸一致；AI 从创作意图产场景 JSON | 真机：同角色道具两镜渲染逐像素核身份一致 |
| **S5 与生成主线衔接** | AI 判断「这镜需要绝对一致/精确机位」时走 Blender，否则走生成模型；缺口诚实标注（D4） | 真机走查：混合时间轴（Blender 片段 + 生成片段）导出成片 |

> 每片做完五门全过（`pnpm run gates`）+ 真机走查（R13，截图人眼判断）才算完。资产生成/渲染的额度花费**默认授权直接跑**，跑完报花销。

## 5. 授权与分发（GPL 安全港，权威源已核，非律师建议）

- **Blender = GPL（分发二进制按 GPLv3+）**。核心规则（blender.org/about/license + FSF GPL-FAQ #MereAggregation，2026-06-21 直读）：
  - **headless 子进程调用（fork/exec + 传文件 + CLI 参数）= arm's-length，不传染**——Nomi 的 Electron/JS 代码可保持闭源。Blender FAQ 逐字允许：「运行于 Blender 之外、不用其源码或 API 调用、为 Blender 生产数据」的软件可自由许可。
  - **不捆绑 Blender 二进制**→ 无 GPL 分发义务。做法 = **检测用户已装 + 调用**（业界默认）。
  - **若写 `import bpy` 的 addon 并分发**→ 那部分须 GPL。故本 lane **不写 addon、不分发 bpy 脚本作为产品**，渲染脚本是 Nomi 内部模板、随主进程跑、不对外分发为独立 Blender 插件。
- **3D 资产 API**：Tripo/Meshy/Rodin 是付费 API，产物授权按各家商用条款（接入时核）。
- **诚实缺口**：无 Blender arm's-length 法院判例（靠基金会+FSF 解释）；socket 桥若传复杂场景态有"语义合并"理论风险——本 lane 用子进程+文件传递正是为避开此风险。

## 7. 风险 / 回滚

| 风险 | 缓解 |
|---|---|
| 渲染慢（Cycles 每帧分钟级） | 范围限短片段/低帧；只在"必须一致+精确机位"的镜用，不普适 |
| 用户没装 Blender | S0 探针 + 引导；没装就此 lane 不可用，不阻塞 Nomi 主流程 |
| 资产质量不够（三角面汤） | 只用作可整体移动的道具/背景，不做变形角色；Tripo 四边面优先 |
| AI 产的场景 JSON 不合理 | 模板化 + 渲染前校验 + 失败透传说人话（不裸写 bpy） |
| 范围膨胀（被诱去做角色动画） | §1.2 硬禁 + §1.3 自检；越界停下上报 |
| **回滚** | 整 lane 是新增 runtime 分路 + 新 archetype kind，与现有生成链解耦；下掉 = 移除该分路，主线零影响 |

## 9. PoC 实测记录（2026-06-21，用户机器 Apple M5 / 24GB / Blender 5.1.2，全程 headless 无 GUI）

用 `/tmp/blender-poc/`（scene.json + render.py / render_orbit.py / render_glb.py）端到端验过 plan 的核心链路。**结论：渲染半段确证可行；资产生成半段（API）未跑。**

### 9.1 已证 ✅
- **架构链通**：`scene.json`（模拟 AI 输出）→ 固定 bpy 模板组装 → `blender -b --python <script> -- <json> <out>` 子进程 → 出图/帧。**AI 产 JSON 不裸写 bpy（§3.1 R2）实测可行**，子进程调用（§3.1 R3）就是 plan 设计的形态。
- **跨镜一致（lane 唯一存在理由）**：同一 subject 三机位渲染，身份/比例/配色逐像素一致，只有相机变。这是扩散视频结构性给不了的。
- **真相机控制**：机位/朝向/焦段全来自 JSON 数值（50mm / 35mm / 低角度），track-to 约束看向目标点。
- **真实 GLB 导入**：Khronos 样例 Duck.glb（非自搭几何体）`import_scene.gltf` 导入 + 包围盒归一化 + 双机位渲染通过——「任意 3D 资产 → headless 渲染」的导入端成立。
- **GPU**：Cycles 跑在 `Apple M5 (GPU - 10 cores)` Metal，自动探测成功。

### 9.2 实测发现，已反哺架构 🔧
- **brew cask 的 Blender 5.1.2 编译时不含 FFmpeg**（`image_settings.file_format` 枚举无 `FFMPEG`，直接渲视频报错）。→ **正确架构 = Blender 只出 PNG 帧序列，视频编码交给 Nomi 随包的 FFmpeg**（实测 `ffmpeg -framerate 24 -i f_%03d.png -c:v libx264` 合成 h264/960×540 成功）。这反而比让 Blender 出视频更干净，且和 Nomi 已带 ffmpeg 一致。**S3 切片按此实现，不依赖 Blender 的 FFmpeg。**
- **渲染就是成本**：3 张 960×540 @ 96 采样 = **75 秒**（GPU）。印证 §2.4「慢」约束——必须限短片段/低采样/低分辨率预览，且只用在"必须一致+精确机位"的镜。
- **取景需自动化**：GLB 导入后默认取景会切边，真实现需自动取景（按包围盒算相机距离/俯仰），列入 S1/S3 实现细节。

### 9.3 未证 ⚠️（诚实标注）
- **文生/图生 3D 资产 API（Tripo/Meshy/Rodin）→ GLB** 这一段**没跑**（需账号/key + 花额度）。是常规 HTTP API + 拿 GLB，风险低但本次未实证。S2 启动时真接真验。
- **绑骨/角色动画**：未试，且 §1.2 明令不做（2026 墙没破）。
- PoC 脚本在 `/tmp`，非仓库代码；正式实现走 plan §4 切片，纳入 runtime 第五条渲染源 + 五门 + R13 走查。
