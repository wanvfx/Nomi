## 0. 一页读懂

**这次要解决的真实摩擦**：预览区现在「剪不顺」——播放器控件单薄（没音量/全屏）、clip 缩略图是假黑底卡、撤销了回不来、裁剪时不知道裁了多少。这些是「不做显得业余、做了用户才敢信」的地基。

**核心判断（被用户测验证过）**：别去和 Premiere/CapCut 拼专业全功能（JKL 穿梭、帧级时间码那些）——**Nomi 用户是 AI 创作者不是剪辑师**，那些对他们是界面噪音（5 画像里新手明确说「JKL 我不知道是啥、帧码我看着晕」）。只补**普通人每天会碰**的那几样，AI 原生的真差异化（原地重生成）在桥方案里做。

## 2. 去向（按 5 画像用户测 + PM 评审裁剪）

| 去向 | 项 | 理由 |
|---|---|---|
| **留（做）** | ① redo 重做 ② trim/split 帧气泡(拖时浮「+12f·00:01:14」) ③ 音量/静音 ④ 全屏 ⑤ 逐帧 ←/→ + Home/End（预览区） | 普通人每天会碰；redo/音量/全屏是明显残缺；气泡是浏览器对手都没做的净增量 |
| **先验证再定** | ⑥ scrub 91ms 卡顿 | 没真机重测、可能不存在；先 probe-latency 实测，真有才修 |
| **移交桥方案** | ⑦ clip 原地重生成（原 B1 = 桥 C1）⑧ 真实帧缩略图（原 A6 = 桥 C3） | 是「画布↔预览桥」的桥动作，统一在 `canvas-preview-linkage.md` 做，不重复 |
| **砍** | JKL·I/O 穿梭、帧级时间码、倍速、掉帧红绿灯 | 剪辑师专业件，Nomi 用户（AI 创作者）当噪音（用户测明确） |
| **延后** | 波纹删除 | 无高频摩擦证据，第二程再看 |

## 4. 范围 / 不动项 / 回滚 / 验收门（R4）

**范围**：§3 三切片。**不引入新框架/技术栈**（R5 不触发）。
**不动项**：① 时间轴本体（轴/trim 小钮/拖动排序/分割/snap/多选）不改，只补；② 不新增设计 token（trim 气泡复用 `--nomi-snap-tag`，音量/全屏控件用 `WorkbenchButton`+`--nomi-accent`）；③ 数据模型 `timelineTypes.ts` 不动（这些都是 UI/控件层）。
**回滚**：每切片独立 commit，新增控件/键位，关入口即回滚。
**验收门**：五门全过；单测（redo 栈、trim 气泡帧数计算）；真机走查（R13 正常 GPU）——redo/音量/全屏/逐帧真点、trim 气泡数字对、scrub 实测延迟；与现状对账（P3）。
