## 两个区域的结构事实

- **剪辑区 = 预览标签**：`PreviewWorkspace` = `TimelinePreview`（播放器 + 控制条）+ `TimelinePanel density="full" showTextTrack`（图片轨/媒体轨/文字轨）。
- **生成画布时间轴**：`GenerationWorkspace` 底部挂同一个 `TimelinePanel density="compact"`（图片轨/媒体轨，**无**文字轨）。
- 二者**共享同一份 `timeline` store** —— 在哪边编辑都改同一条时间轴。

## 问题清单（分级）

### P0 — 数据损坏（已修 + 推送 5e537e7）
**#1 文字 clip 跨会话撞 id → 编辑一条串改另一条。**
- 真机复现：新建字幕 → 改字 → 时间轴上另一条旧字幕文字也被同步改成同一份。
- 根因：`timelineTextEdit.ts` 的 `createTextClipId` 用**每次加载归零的自增序列**（`text-1/2…`）。落盘旧 clip 反序列化仍带上一会话 `text-1`，新建时序列从 0 重数 → `text-1` 撞旧 clip。`updateTextClipText` 的 `.map` 同时命中两条、React key 也重复。
- 修法（根除整类）：① id 改 `crypto.randomUUID()`（仓库既有范式）；② `normalizeTimeline` 加载时把重复 id 重铸唯一 id → 已损坏工程自愈；③ 2 条回归断言。
- 同类排查：全仓 reset-counter id 只有 `textClipSeq` 既持久化又归零，其余（toastSeq/attachmentSeq/eventSeq）均不落盘，安全。

### P1 — 交互摩擦（待拍板）
**#2 时间轴编辑无键盘 Undo。** 真机 Cmd+Z 不撤销 split（撤销疑似只在某些操作有 toast 入口）。分割/裁剪/移动这类破坏性编辑没有顺手的撤销，误操作只能手动还原。

**#3 在 clip 中段分割很别扭。** 点 clip 永远把 playhead 跳到 **clip 起点**；要在中段切，必须「选中 → 再去标尺 scrub 到中段（还要避开吸附到边缘）→ 再按 ✂/S」。分割工具在时间轴右上角、离目标 clip 远，且**没有「将在此处切」的预览竖线**，盲切。

**#4 文字 clip 可完全重叠堆同位。** 多条字幕落在同一帧会层叠在同一像素位置，难以单独点选/区分（真机清理时遇到两条 clip cx 完全相同）。

### P2 — 架构/性能（待拍板）
**#5 两个完整 TimelinePanel 同时挂载。** 预览（full）与生成（compact）各挂一个 `TimelinePanel`，隐藏的那个 clip 仍在 DOM（w:0）。同一 store 双订阅、双份轨道 DOM、双份 window keydown 监听。功能无碍但是重复渲染面。

### P3 — 视觉/文案（待拍板，需设计样张）
**#6 预览控制条会被撑很宽。** 选中文字后追加「字号 −100+ / 字体 默认黑体▾」控件，1440 窗口下控制条已 895px；子项全 `shrink-0`，窄窗口（≈<960px）右端「导出 MP4」有被挤出/截断风险（溢出而非换行）。

**#7 轨道命名不一致。** 「图片轨」（具体）vs「媒体轨」（笼统）—— 视频轨却叫「媒体轨」；纯图片项目里「媒体轨」永远空占一行 52px。

**#8 时间轴区域贴窗口底边。** 预览时间轴 `regionBottom 828 ≈ winH 831`（仅 3px）；文字轨/底部 clip 离窗口下边极近，呼吸感不足。

**#9 clip 缩略图辨识度低。** 宽参考图 `object-cover` 后只见局部（如「创作者」缩略图=几个小人剪影），一眼认不出是哪一镜。
