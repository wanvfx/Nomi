## 1. 架构 / 机制层根因（重）

### A1 · 节点参数面板用「离屏渲染全部模型」测最宽卡宽 〔F7｜性能+A11y〕
- **现象**：选中一个图片节点，实测渲染 **7 个「锁定节点」按钮**（1 真 + 6 个隐藏模型克隆）、28 个 button 挂在 `left:-99999px`。每模型一份完整 footer（锁定徽章+参数条+生成钮）。
- **根因**：`src/workbench/generationCanvas/nodes/NodeComposerWidthMeasurer.tsx:60-66` 离屏容器 + `modelOptions.map` 把**每个候选模型的真实交互 footer** 渲染到屏外测 `max(scrollWidth)`；挂载点 `NodeGenerationComposer.tsx:260`（每选中一个节点就挂一个测量器）。宽度回流：`NodeGenerationComposer.tsx:141-148,178`。
- **代价**：① 隐藏 DOM = 选中节点数 × 模型数，随模型/节点增长劣化；② A11y：克隆里的 `NodeLockBadge.tsx:12`、`InlineParameterBar.tsx:44,64` 是裸 `<button>`，**无 `tabIndex=-1`/`inert`**，仅靠容器 `aria-hidden+visibility:hidden`(`:62-64`) 抑制 → 键盘 Tab/读屏可落到 -99999 的幽灵控件。
- **地基问题**：卡宽本应由内容/设计 token 决定，不该靠「渲染全模型集群实测」。来源 commit `e26d240`。
- 分层：症状=7 个锁钮；根因=离屏全模型实测的设计；入口集=每个被选中的生成节点。

### A2 · 创作选区「生成图片」不写剧本节点 → 文本双真相源 〔F9｜数据一致性〕
- **现象**：创作区选中脚本→气泡「生成图片」，文本被塞进**新图片节点的 prompt**，而画布既有的「剧本片段」文本节点仍空（占位「在这里写文本……」）。同一段源文本散落两处、无单一真相源。
- **根因**：`src/workbench/creation/SelectionGeneratePopover.tsx:102-106,210,226` 只硬编码 `kind='image'|'video'`；`createNodeFromSelection.ts:22-35` 把选区文本当 `prompt` 直接 `addGenerationNode`，**从不写入/链接剧本片段节点**。文本节点真相源在 `generationCanvasDefaults.ts:7-19`（`gen-v2-text-1` 剧本片段）却被该流程绕过。存在 text 路径(`creationNodeCommands.ts:17-27`)但只接在文档工具(`WorkbenchEditor.tsx:149`)，气泡不走。
- **冲突**：与 memory「分镜方案文档主链路 文本→剧本片段→画布锚」单源目标直接相悖。
- 分层：症状=剧本节点空、文本进图片 prompt；根因=气泡流程的路由决策；入口集=气泡生成图片/视频两个按钮。

### A3 · 空白项目预置「已编号空节点对」→ 镜号跳跃 〔F12｜可理解性〕
- **现象**：空白项目其实预置 镜头1 剧本片段(空)+镜头2 关键画面(空)；生成图片再加镜头3，编号跳、空实混杂。
- **根因**：`generationCanvasDefaults.ts:6-41` 默认快照就含两节点+一条边，关键画面 `shotIndex:1`；`canvasNodeActions.ts:44-46` + `shotNumbering.ts:33-39` 新图片取 `nextShotIndex`=已占用最大+1，**空种子节点先占了号**。

### A4 · 新建节点恒落中心、无碰撞避让 〔F10｜布局〕
- **现象**：生成图片 / 添加 3D 场景等新节点恒落画布中心，叠在既有节点上。
- **根因**：`GenerationCanvas.tsx:655-673` 锚点硬编码视口 `0.38×0.42`，所谓避让只用**整数像素点等值**判断(`occupied` set)，错开 5px 或真实包围盒重叠完全检测不到 = 形同虚设；`canvasNodeActions.ts:37-38` fallback 仅按计数斜向 `+34/+30` 级联。**全程无 AABB/网格吸附**（memory「网格避让 D」确认仍缺，只在 agent 批量路径修过 origin）。

### A5 · 分镜自动布局：镜序≠空间序 + 异高节点重叠 〔F14/F15｜布局〕
- **现象**：示例「30 秒产品介绍」底排左→右是 镜头6/镜头5/镜头7（6 在 5 前）；图片节点镜头4(h≈240) 卡在两排文本(h≈164)之间，实测与 镜头5/6 重叠 2 处。
- **根因**：① `storyboardPlan.ts:173-205` 按数组下标布局，`shot.index` 只用于标题(`:194`)**从不排序**；`trajectoryLayout.ts:74-78` 网格行列=下标的纯函数 → LLM 乱序就照排。② `trajectoryLayout.ts:64-73` 网格 fallback 用**统一 cellHeight**(全批 max+GAP)，忽略每节点真实盒高；分层路径(`:81-102`)能按各自高度堆叠，但只在 `distinctLayers>=2 且无 null layer` 时触发，混入 text/output(null layer) 即落回统一网格 → 高图片节点溢出行距、压到邻居。

### A6 · 选中节点浮动 composer 被画布顶边裁切、无 clamp 〔F11｜遮挡〕
- **现象**：节点靠画布顶时，参数 composer 卡定位到 y≈-32（顶边之上）被裁；密集网格下还压住邻居。
- **根因**：`NodeGenerationComposer.tsx:126-130` 翻转判定 flipUp 只要 `spaceAbove>spaceBelow`，**从不校验 `spaceAbove>需要高度`**，也无视口 clamp；向上偏移 `:162-164` 无上限；`generationCanvas.css:4` `.__stage{overflow:hidden}` 且 composer 是节点子元素(`BaseGenerationNode.tsx:880`)**无 BodyPortal** 逃逸 → 翻到顶外即被裁。既有遮挡防线只解决了底边/时间轴向，顶边对称情形未处理。

### A7 · 项目创建契约不统一 → 库无限膨胀 〔F1/F6｜数据治理〕
- **现象**：磁盘 81 项目（30 示例含 18 重复 + 51 未命名）。
- **根因（已部分修）**：示例重复 seeding 历史根因=`createLocalProject`(`projectRepository.ts:108-151`) 无条件铸新 id、名字非身份；触发集=空库 CTA + 模型接入后自动重放(`NomiStudioApp.tsx:354-368`)。**现已加 seedKey 幂等守卫**(`NomiStudioApp.tsx:309-325`)。
- **根因（未修）**：① 历史 81 个磁盘记录**无去重/迁移**；② 空白项目持久化即落盘(`projectRepository.ts:143-149`)、默认名仅时间戳(`:46-48`)、**全仓无任何空项目回收/GC**（grep recycle/gc/cleanup/purge 皆空）→ 每次「新建空白」永久留一条未命名记录（已积 51 条）。

### A8 · 示例项目零渲染产物 → 库缩略图全空 〔F2｜产品/onboarding〕
- **现象**：打开任一「示例」直接进画布，镜头全是空 hatched「等待生成」；库卡片几乎全是灰占位。
- **根因**：`tryExample`(`NomiStudioApp.tsx:340-347`) 只 `buildStoryDocument` 不自动拆镜/不出图（避免偷耗额度，合理）；缩略图**只**来自画布节点的 `result.url`(`projectNormalize.ts:15-26`)，示例无产物 → 永远空；**示例不附任何静态预览图**。⇒ 新用户眼里「示例」= 一堆空格子，完全没起到「展示成果」的作用。

## 2. 设计 / UX 层（眼判）

### D1 · 选区气泡菜单与固定工具栏重复 〔F5〕
选中文字弹出的 bubble 含 加粗/斜体/H1/H2（与顶部固定工具栏**完全重复**），仅「生成图片/生成视频」是差异化动作；bubble 在 y=123、固定栏 y=101 仅隔 22px，视觉叠成两层。R2 极简：bubble 应只留生成动作。

### D2 · 项目库卡片信息无区分度 〔F3〕
卡名多为「未命名项目 06/12 00:03」+ 相对时间，51 个未命名彼此无法分辨；缩略图又全空(A8) → 整个库**无任何可识别锚点**，用户找不回自己的项目。

### D3 · 非空库无冷启动引导入口 〔F4〕
83 项目时首页无「30 秒体验」入口（仅空库 hero 有），老用户回不到引导旅程。待确认是否设计如此。

### D4 · 自动建节点的归属/编号难追踪 〔F12，与 A3 同源〕
空种子节点 + 跳号 + 空实混杂，用户分不清哪个节点承载自己的内容（设计可理解性问题，根因见 A3）。

### D5 · 密集网格下选中即遮挡邻居 〔F11，与 A6 同源〕
选中中部节点，其 composer 弹出压住上/左/右邻居；在示例这种 3×N 密排下尤其明显。

### D6 · 示例镜序空间错位读不出故事顺序 〔F14，与 A5 同源〕
眼看底排 6→5→7，用户无法按空间顺序读出分镜先后。

### D7 · 模型接入卡底部小灰说明像一堵 fine-print 墙 〔观察〕
APIMart 展开后底部多行小灰字说明，信息密但行动价值低，建议按 R2 审视删减（待逐字核对内容）。
