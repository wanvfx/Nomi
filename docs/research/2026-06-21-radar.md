## 一句话总结

学术界正用 **DirectorBench / EntityBench / NarrLV** 给「decompose-stitch 影视生成」打分——**Nomi 就是这个架构**，且这类系统的公认软肋（镜间转场、跨镜身份）正是 Nomi 的软肋。本轮最该动的两件事：

1. **接 DirectorBench / EntityBench 的「镜间」维度进 eval 体系**——它指认转场质量（行业平均 0.256）和跨镜身份是中心瓶颈，Nomi 现有 eval 多在「单镜内」打转，这块是盲区。（对标基准 🔵）
2. **给跨镜身份补一个「记忆库」层**——VideoMemory 的「每镜后更新角色/道具/背景描述符」几乎是你记忆系统重设计的同款思路，可直接借鉴落到 `reconcile.ts` + 设定卡。（可落地 🟢）

## 🟢 可落地（这周/这月能进 Nomi）

### VideoMemory：用「动态记忆库」锁跨镜身份（2026-01）
[arxiv 2601.03655](https://arxiv.org/abs/2601.03655v1) · 多 agent 把叙事拆成镜头，**Dynamic Memory Bank 存角色/道具/背景的显式视觉+语义描述符，每镜后更新**——既保身份又允许剧情驱动的变化（换衣/受伤），跨远镜检索复用。
- **落到哪**：`src/workbench/generationCanvas/agent/reconcile.ts` + 设定卡（`CharacterCardNode.tsx`）。Nomi 现在身份靠 character_ref 边喂参考图；缺的是「跨镜的、可随剧情更新的描述符库」。这正是你 `docs/plan/2026-06-20-memory-system-redesign.md`「设定卡库」的学术背书。
- **怎么改**：设定卡增加「随镜次演进的状态字段」（如 shot 5 起换装），生成时按 shotIndex 取当前态喂参考，而非全程一张静态图。

### RAPO / VPO：结构化 prompt 改写补齐糙输入（2025-03/04，training-free 分支可直接用）
[RAPO 2504.11739](https://arxiv.org/html/2504.11739v1) · [VPO 2503.20491](https://arxiv.org/abs/2503.20491) · 用户输入糙/短，但 T2V 模型训练时吃的是高度结构化的长描述——这个 gap 直接拖垮生成质量。RAPO 双分支（关系图取修饰词对齐训练格式 + LLM 重写）。
- **落到哪**：`electron/promptLibrary/` + AI 优化节点 + `electron/ai/promptSanitize.ts`。Nomi 已有「AI 优化」按钮，但策略可对齐 RAPO 的「对齐训练 prompt 结构」而非自由改写。
- **怎么改**：AI 优化下沉一套「按目标模型 archetype 的结构化模板」（镜头/主体/动作/光/运镜分槽），符合 P4 通用——档案声明结构，通用系统填。

### PhyPrompt：零样本跨模型的 prompt 精炼（2026-03，zero-shot transfer）
[arxiv 2603.03505](https://arxiv.org/html/2603.03505v1) · RL 训出的 prompt rewriter，**zero-shot 跨多个异构 T2V 生成器不用重训**——符合 Nomi「不为不同模型写两套」的 P4。物理合理性奖励对「动作类」镜头（DirectorBench 指认最难，0.455）特别有用。
- **落到哪**：同上 prompt 优化层；先借其「时间相关奖励：先语义对齐后物理常识」的思路，不必自己训 RL。

## 🟡 架构教训（不直接落地，但影响怎么设计）

### Co-Director：waterfall 流水线的级联误差（2026-04）
[arxiv 2604.24842](https://arxiv.org/html/2604.24842v1) · 直接批判 decompose-stitch：① 级联误差（早期错向下传，毁长程一致）② 子 agent 各读静态模板→「语义碰撞」、成片散 ③ 信用分配难（终端失败查不到是哪个 prompt）。解法：别逐模块调，用**中心参数自顶向下统一创作意图** + **分解奖励**（每轴算分，绕开信用分配）+ **局部自纠环**（关键帧批评环，去掉它质量掉 9.8 分）。
- **对 Nomi 的影响**：直击你「拆镜头两套矛盾默认」的根（记忆 two-agents-persona-ux-audit）。教训：**统一意图应自顶向下注入各 agent，不是每个 agent 各自解读**；关键帧应**联合评估**抓序列级漂移，不是逐帧看。eval 体系若上奖励信号，要按维度分解而非单一标量。

### A2RD / HoloCine：长视频一致性的新范式（2026-05/2025-10）
[A2RD 2605.06924](https://arxiv.org/html/2605.06924) · [HoloCine 2510.20822](https://arxiv.org/abs/2510.20822) · 「holistic 整体生成多镜」vs Nomi 的「逐镜拼」。短期 Nomi 受限于所接的第三方模型（Seedance/Sora 单镜），但这是方向信号：**整体生成正在取代纯 stitch**——盯所接 vendor 何时出多镜原生能力。

## 🔵 对标基准（接进 eval 体系自评 Nomi）

### DirectorBench：直接给 ViMax（=Nomi 同架构）打了分（2026-05）
[arxiv 2605.30090](https://arxiv.org/html/2605.30090) · 5 维 40 检查点（剧本叙事/视觉/音频/跨模态/稳定性），多 agent 诊断定位 checkpoint 级瓶颈。**关键发现**：decompose-stitch 系统「几乎所有连贯性子指标都差」，ViMax temporal coherence 仅 **0.18**；**镜间转场是全行业中心瓶颈**（均 0.256，最好 Kling 0.356）；单镜内反而高（需求满足 0.71、文-视一致 0.63）。MovieAgent「用户需求一致性极弱 0.08」。
- **接法**：把「镜间转场质量 / 跨镜身份 / 跨模态唇形对齐」三个维度抄进 `docs/plan/2026-06-11-eval-system-master-plan.md` 的评测面——这是 Nomi eval 现在的盲区（多在单镜内）。动态检查点（有对话才触发唇形检查）的设计也值得借。

### EntityBench：跨镜多实体一致基准（2026-05）
[arxiv 2605.15199](https://arxiv.org/html/2605.15199) · 140 集 ~2491 镜，显式「实体排期表」标每镜该出现哪些角色/物/景，**同时评多实体一致**（不是单主体）。
- **接法**：Nomi 拆镜头已产 shot 列表，可派生「实体排期表」做自检；身份测试从单角色扩到多角色同框（记忆 Identity-GRPO 也说多角色互动最难）。

### NarrLV：叙事丰富度量化（2025-07，仍现役）
[arxiv 2507.11245](https://arxiv.org/html/2507.11245) · 受影视叙事理论启发，用 **Temporal Narrative Atom（TNA）计数**量化叙事丰富度，与人评高度一致。
- **接法**：Nomi eval 的「主观人锚」层可用 TNA 思路把「这片讲清楚故事了吗」半客观化。
